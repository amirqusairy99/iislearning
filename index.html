Introduction to IIS

Internet Information Services (IIS) is Microsoft's powerful, flexible web server built into Windows. It hosts websites, web applications, REST APIs, and services on the Windows platform.

If IIS is installed on your machine, you are close to ready. Verify that the needed features are enabled before you start hosting sites or apps.

What Can IIS Do?

  • Host static websites (HTML, CSS, JavaScript, images)
  • Run ASP.NET and ASP.NET Core web applications
  • Serve PHP, CGI, and other server-side technologies
  • Handle FTP and SMTP services
  • Act as a reverse proxy for other services
  • Enforce SSL/TLS encryption (HTTPS)

IIS vs Other Web Servers

IIS is Windows-native and integrates deeply with Active Directory, .NET, and Windows authentication. While Apache and Nginx dominate Linux, IIS is the go-to for Windows enterprise environments.

# Check if IIS is running via PowerShell Get-Service -Name W3SVC # W3SVC = World Wide Web Publishing Service (the IIS engine)

Current IIS Version

Modern Windows client and server environments commonly use IIS 10.x. Exact capabilities depend on your Windows version and patch level, so treat the OS build as the source of truth when checking feature availability.

Where is it? Press Win+R, type inetmgr, press Enter to open IIS Manager.

The Request Processing Pipeline

When a browser makes a request, it passes through several layers before a response is sent. Understanding this pipeline helps you configure, debug, and optimize IIS effectively.

HTTP.sys - The Kernel-Mode Driver

HTTP.sys is a Windows kernel-mode driver that listens for HTTP/HTTPS requests. It handles listening on ports, SSL termination, response caching, and request queuing for application pools.

# IIS Request Flow Browser Request | HTTP.sys (Kernel Mode) <-- Handles TCP/IP, SSL, caching | W3SVC Service <-- Manages app pools, config | w3wp.exe Worker Process <-- One per Application Pool | IIS Modules + Handlers <-- Auth, compression, rewrite... | Your App Code <-- ASP.NET, PHP, static files | HTTP Response --> Browser

Worker Processes (w3wp.exe)

Each Application Pool runs in its own w3wp.exe worker process. This isolation means one crashing app does not take down others. You can see them in Task Manager.

Pipeline Modes

Integrated vs Classic: Always use Integrated mode for new apps. Classic is only for legacy apps that require it.

You Already Have IIS!

Since you have IIS installed, verify the right features are enabled. IIS is modular - you only install what you need.

Method 1: Windows Features (GUI)

Go to Control Panel → Programs → Turn Windows features on or off. Expand Internet Information Services and check the features you need.

Key features to enable:
  • Web Management Tools → IIS Management Console
  • WWW Services → Application Development (ASP.NET)
  • WWW Services → Common HTTP Features
  • WWW Services → Security

Method 2: PowerShell (Run as Admin)

# Windows Server Install-WindowsFeature -Name Web-Server -IncludeManagementTools # Windows 10/11 Desktop Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServer -All # Check what IIS features are installed Get-WindowsFeature | Where-Object {$_.Name -like "Web-*" -and $_.Installed}

Verify IIS is Working

# Check service status Get-Service W3SVC # Start IIS if stopped Start-Service W3SVC
Quick Test: Open your browser and go to http://localhost. If you see the IIS welcome page - it is working!

Default Paths

  • Web root: C:\inetpub\wwwroot\
  • Config: C:\Windows\System32\inetsrv\config\
  • Logs: C:\inetpub\logs\LogFiles\

Opening IIS Manager

IIS Manager is your control center. Press Win+R, type inetmgr, press Enter. Or search "IIS Manager" in Start.

The Three-Panel Layout

  • Left (Connections) - Tree: Server → Sites → Applications → Virtual Directories
  • Center (Features) - Icons for configuring the selected node
  • Right (Actions) - Context-sensitive actions

The Connections Tree

SERVER-NAME +-- Application Pools | +-- DefaultAppPool | +-- .NET v4.5 Classic +-- Sites +-- Default Web Site (port 80) +-- aspnet_client +-- (your apps)

Key Feature Icons

  • SSL Settings - Configure HTTPS requirements
  • Authentication - Enable/disable auth methods
  • Default Document - Set index.html, default.aspx etc.
  • Directory Browsing - Show file listings (off by default)
  • Compression - GZip static/dynamic content
  • URL Rewrite - Redirect and rewrite URLs
Be careful: Changes save to C:\Windows\System32\inetsrv\config\applicationHost.config. Back this up before major changes.

What is a Binding?

A binding tells IIS how to listen for incoming requests. Each binding is: Protocol + IP Address + Port + Host Name.

# Example bindings: http *:80: <-- All IPs, port 80, any hostname http *:80:mysite.local <-- Port 80, only hostname "mysite.local" https *:443: <-- HTTPS on all IPs http 192.168.1.10:8080: <-- Specific IP, port 8080

Creating a New Website

In IIS Manager: Right-click Sites → Add Website. Fill in: site name, application pool, physical path, and binding.

Multiple Sites on Port 80

Edit the hosts file at C:\Windows\System32\drivers\etc\hosts (open Notepad as Admin):

127.0.0.1 site1.local 127.0.0.1 site2.local 127.0.0.1 myapp.local

Then in IIS, set each site's binding host name accordingly. IIS routes by the HTTP Host header.

PowerShell: Create a Site

Import-Module WebAdministration New-WebSite -Name "MyApp" -Port 8080 -PhysicalPath "C:\inetpub\myapp" # List all sites Get-WebSite
Port conflict? Run netstat -ano | findstr :80 in cmd to find what is using port 80.

What is an Application Pool?

An Application Pool isolates web apps in their own worker process (w3wp.exe). If one app crashes, others are unaffected.

.NET CLR Version

  • v4.0 - for .NET Framework 4.x apps
  • v2.0 - for .NET Framework 2.0/3.5 apps (legacy)
  • No Managed Code - for PHP, static files, Node.js via iisnode

Identity (Security Account)

ApplicationPoolIdentity <-- Default, most secure (virtual account) LocalService <-- Low-privilege Windows service account NetworkService <-- Can access network resources LocalSystem <-- Full admin - NEVER use in production! Custom Account <-- Specific domain or local account

Recycling

IIS automatically restarts worker processes to keep apps healthy. Configure in IIS Manager → App Pools → Advanced Settings → Recycling.

# PowerShell: Create an app pool New-WebAppPool -Name "MyAppPool" # Restart an app pool Restart-WebAppPool -Name "MyAppPool"
Best Practice: Give each application its own dedicated app pool for isolation.

What is a Virtual Directory?

A Virtual Directory maps a URL path to a physical folder on disk (or a UNC network path), letting you serve content from different locations under one website.

# Example mappings: URL: http://mysite.com/images ---> D:\SharedAssets\images\ URL: http://mysite.com/docs ---> \\fileserver\documents\

Virtual Directory vs Application

  • Virtual Directory - just a path mapping, shares the parent app pool
  • Application - a virtual directory with its own app context, app pool, and web.config

Creating a Virtual Directory

In IIS Manager: expand your site → right-click → Add Virtual Directory. Enter an alias (URL segment) and the physical path.

# PowerShell New-WebVirtualDirectory -Site "Default Web Site" -Name "images" -PhysicalPath "D:\assets\images" Get-WebVirtualDirectory -Site "Default Web Site"
UNC Paths: If using a network path like \\server\share, provide credentials via "Connect As" - the app pool identity cannot authenticate to network shares by default.

Why HTTPS?

HTTPS encrypts traffic between browser and server, prevents eavesdropping, and is required for many modern browser features. You enable it in IIS with an SSL/TLS certificate.

Certificate Options

  • Self-signed - free, for local dev only (browsers will warn)
  • Let's Encrypt - free, trusted, 90-day auto-renewal (use win-acme for IIS)
  • Commercial CA - DigiCert, Comodo, etc. (for production)

Step 1: Create a Self-Signed Certificate

# PowerShell (run as Admin) New-SelfSignedCertificate -DnsName "localhost", "mysite.local" -CertStoreLocation "cert:\LocalMachine\My" -NotAfter (Get-Date).AddYears(2)

Step 2: Bind Certificate to IIS

In IIS Manager: select your site → Bindings → Add → type https, port 443, select your certificate from the dropdown.

Step 3: Force HTTPS Redirect (web.config)

<configuration> <system.webServer> <rewrite> <rules> <rule name="HTTP to HTTPS" stopProcessing="true"> <match url="(.*)"/> <conditions> <add input="{HTTPS}" pattern="^OFF$"/> </conditions> <action type="Redirect" url="https://{HTTP_HOST}/{R:1}"/> </rule> </rules> </rewrite> </system.webServer> </configuration>
HSTS Tip: On newer IIS 10 environments you may be able to use built-in HSTS support. Otherwise, add the Strict-Transport-Security response header manually after you have confirmed HTTPS works correctly.

IIS Authentication Types

Configure them in IIS Manager under Authentication.

Anonymous Authentication

Default for public websites. Anyone can access without credentials. Requests run as the IUSR account.

Windows Authentication

Uses Windows credentials (Active Directory or local accounts). Great for intranet apps. Uses NTLM or Kerberos - domain users authenticate automatically, no password prompt.

# Enable Windows Auth via PowerShell Set-WebConfigurationProperty -Filter "/system.webServer/security/authentication/windowsAuthentication" -Name enabled -Value True -PSPath "IIS:\Sites\MyApp"

Basic Authentication

Prompts for username and password. Credentials are Base64 encoded - always use with HTTPS! Without HTTPS, credentials are visible in plain text.

IP Address Restrictions

<!-- web.config: Allow only specific IPs --> <security> <ipSecurity allowUnlisted="false"> <add ipAddress="192.168.1.0" subnetMask="255.255.255.0" allowed="true"/> <add ipAddress="127.0.0.1" allowed="true"/> </ipSecurity> </security>
Rule: Never use Basic Authentication over plain HTTP. Always pair it with HTTPS.

What is URL Rewrite?

The URL Rewrite Module transforms incoming URLs before IIS processes them. Essential for SEO-friendly URLs, HTTP to HTTPS redirects, and reverse proxy setups.

Prerequisite: Install the IIS URL Rewrite Module from the official IIS/Microsoft download page if it is not already present on the server.

Rule Structure

Match: Pattern to test against the URL (regex or wildcards) Conditions: Optional extra checks (HTTPS status, host, query string) Action: What to do - Rewrite, Redirect, Forbidden, etc.

1. Redirect www to non-www

<rule name="www to non-www" stopProcessing="true"> <match url="(.*)"/> <conditions> <add input="{HTTP_HOST}" pattern="^www\.(.*)"/> </conditions> <action type="Redirect" url="https://{C:1}/{R:1}" redirectType="Permanent"/> </rule>

2. SPA Routing (React / Angular)

<rule name="SPA Routes"> <match url=".*"/> <conditions logicalGrouping="MatchAll"> <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true"/> <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true"/> </conditions> <action type="Rewrite" url="/index.html"/> </rule>

3. Reverse Proxy to Node.js

<rule name="Node.js Proxy"> <match url="(.*)"/> <action type="Rewrite" url="http://localhost:3000/{R:1}"/> </rule>
Remember: Rewrite = URL changes internally, browser URL stays the same. Redirect = browser URL changes (301/302 response).

IIS Log Files

IIS logs every request to text files at: C:\inetpub\logs\LogFiles\W3SVC1\ (number matches your site ID).

Log Entry Example

#Fields: date time s-ip cs-method cs-uri-stem s-port c-ip sc-status sc-substatus time-taken 2024-01-15 14:23:01 192.168.1.1 GET /index.html 80 10.0.0.5 200 0 15 2024-01-15 14:23:05 192.168.1.1 GET /missing.html 80 10.0.0.5 404 0 8

Key Log Fields

  • sc-status - HTTP status (200=OK, 404=Not Found, 500=Server Error)
  • sc-substatus - IIS sub-status for more specific error info
  • time-taken - request duration in milliseconds
  • cs-uri-stem - the URL path requested
  • c-ip - client IP address

Failed Request Tracing (FREB)

Detailed diagnostics for failed requests. Enable in IIS Manager: site → Failed Request Tracing Rules. Logs go to C:\inetpub\logs\FailedReqLogFiles\.

Real-time Monitoring

# Watch IIS logs live in PowerShell Get-Content "C:\inetpub\logs\LogFiles\W3SVC1\u_ex*.log" -Wait -Tail 20 # Check active worker processes Get-Process w3wp | Select Id, CPU, WorkingSet
IIS Manager tip: Check Worker Processes under the server node for real-time CPU/memory usage per app pool.

Performance Optimization

1. Enable Compression

# Enable GZip compression via PowerShell Set-WebConfigurationProperty -Filter "system.webServer/httpCompression" -Name doDynamicCompression -Value True Set-WebConfigurationProperty -Filter "system.webServer/httpCompression" -Name doStaticCompression -Value True

2. Static File Caching (web.config)

<staticContent> <clientCache cacheControlMode="UseMaxAge" cacheControlMaxAge="7.00:00:00"/> </staticContent>

3. Recycle Scheduling

Avoid recycling during peak hours. Set a specific recycling time (e.g. 2 AM) in App Pool → Advanced Settings → Recycling.

Common Troubleshooting

HTTP 403.14 - Directory Browsing Denied

No default document found. Add index.html or default.aspx as a Default Document in IIS Manager.

HTTP 500.19 - Config Error

Usually a malformed web.config. Validate XML syntax and check for locked sections in applicationHost.config.

HTTP 503 - Service Unavailable

# App pool is stopped. Restart it: Start-WebAppPool -Name "DefaultAppPool" # Check Event Viewer > Application log for crash details

Permission Errors

# Grant app pool identity access to a folder icacls "C:\inetpub\myapp" /grant "IIS AppPool\DefaultAppPool:(OI)(CI)RX"
